# Основная концепция
**Solas** — это open-source игровой движок, написанный на .NET 10 и ориентированный прежде всего на инди-разработчиков. Его главный принцип можно выразить одной фразой из манифеста проекта: _"# Создавайте геймплей, а не боритесь с архитектурным кодом."_ Каждое архитектурное решение в Solas принято так, чтобы убрать лишнее трение между идеей и её воплощением в коде.

## Философия движка
Solas строится на двухуровневой системе приоритетов:

- **DX (Developer Experience)** — главный приоритет архитектора движка. Код должен быть читаемым, предсказуемым и легко тестируемым.
- **UX (User Experience)** — главный приоритет разработчика игры. Синергия этих двух ценностей позволяет командам создавать выдающиеся игры с максимальной скоростью.

  > **Solas** изначально спроектирован с учётом командной работы. Его архитектура естественным образом разделяет зоны ответственности, что практически исключает конфликты при слияниях в Git и упрощает CI/CD-автоматизацию.

## Ключевые возможности
- **EDL-паттерн** — уникальная архитектура Entity–Data–Logic, объединяющая лучшие стороны OOP, COP и ECS.
- **Реактивные данные** — изменения состояния распространяются автоматически через `ReactiveProperty<T>` и `DataProperty<T>`.
- **Встроенный DI** — надёжная система внедрения зависимостей без сторонних библиотек.
- **Многопоточность** — нативная поддержка параллельных вычислений через Task Parallel Library .NET 10.
- **Виртуальная файловая система (VFS)** — абстракция над путями к ресурсам с поддержкой монтирования (`Mount`).
- **Spaces (пространства)** — иерархическая система управления игровым миром с глобальными и локальными пространствами.
- **Source Generators** — кодогенерация на этапе компиляции снижает количество бойлерплейта.
- **Git-friendly архитектура** — сериализация только необходимого в любом в формате.

## Архитектурный паттерн EDL
EDL расшифровывается как **Entity – Data – Logic**. Это три несмешиваемых слоя, из которых собирается любой игровой объект.

| Слой | Класс / Интерфейс | Роль |
| --- | --- | --- |
| **Entity** | `Entity` | Легковесный контейнер с уникальным `Guid` и метаданными. Не содержит логики. |
| **Data** | `IData` | Чистое состояние без какой-либо логики. Реагирует на изменения через реактивные свойства. |
| **Logic** | `Logic` | Ультралёгкий класс поведения, хранящий только ссылку на `Entity` и работающий с её `Data`. |

```csharp
// Entity — empty container
var enemy = new Entity();

// Data — state only
public class EnemyData : IData
{
    public DataProperty<int> Health = new();
    public DataProperty<float> Speed = new();
}

// Logic — behavior only
public class EnemyAI : Logic
{
    // Entity is accessible via the Entity property (inherited from Logic)
    public void Update()
    {
        var data = Entity.GetData<EnemyData>();
        // work with data.Health, data.Speed, etc.
    }
}

// Assemble the object
var data = enemy.AddData(new EnemyData());
data.Health.Value = 150;
enemy.AddLogic<EnemyAI>();
```

  Такое разделение гарантирует, что ваши данные можно сериализовать и протестировать независимо от логики, а логику — подключить к любой сущности без дублирования кода.

# Serialization
Сериализации в движке уделено большое внимание. Мы сделали её максимально гибкой. 
## Serializer
Для сериализации примитивов используется абстрактный класс `Serializer`, он может быть только 1 на всю игру. В ядре движка уже есть два сериалайзера: Json и Binary. По умолчанию используется Json, при сборке все данные переводятся в Binary. 
Сериалайзер по умолчанию можно изменить в \*.csproj вашего проекта. 
Сериалайзер для сборки в файле настроек сборки.
Однако вы можете не ограничиваться в стандартных сериалайзерах и написать свой, наследуемый от `Serializer`!
## Кастомная сериализация данных
Порой стандартной сериализации данных бывает недостаточно, что делать?
Solas предлагает элегантное решение этой проблемы – кастомные сериалайзеры. Они представляют собой обертки над определенными нестандартными типами. Когда такой сериалайзер появляется в проекте, генераторы исходного кода сразу же подхватывают его и не генерируют стандартный.
> Стандартные сериалайзеры генерируются только для `IData` и инджектируемой `Logic`.

Чтобы написать свой кастомный сериалайзер достаточно создать класс, который будет наследоваться от `ICustomSerializer` и в скобках указать тип, для которого пишется сериализация. Далее необходимо непосредственно в функциях записи и чтения логику сериализации. 
```csharp
public void Write(MyType value, FileStream stream, Serializer serializer, string name = null)
{
	//TODO
}
public MyType Read(FileStream stream)
{
	//TODO
}
```
В функции записи есть аргумент сериалайзера, именно его необходимо использовать, так как при сборке сериалайзеры могут поменяться и если использовать стандартный, то это может привести к ошибкам. 
В функции чтения просто берите сериалайзер из `Query.Serializer`
